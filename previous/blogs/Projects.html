    A weekend of progress

    JUL. 14, 2020

    Last week was the first 4 day work week of this month and it was really, really good. I managed to be apart of my very first release into a live server in work and with the Friday off I managed to get through some of my projects to do list. Not only that I was productive throughout the weekend and still managed to get a nice balance between being productive and doing the things I like to do to relax. This is the most refreshed I've felt from a weekend in a long time. If I'm feeling like this now I hope this is a sign of good things to come.

    Friendship ended with i3, DWM is my best friend now

    I've been meaning to try out a new window manager for my ThinkPad for about over a month now but after work I didn't really make the time to sit down and install a new one. Well on the weekend I finally got around to trying DWM and honestly it's my new favourite thing. I've been using i3 for a while now, I think I installed it back in January of this year? i3 is a nice window manager to use for getting into them and I had some good use out of it but I think around March I started feeling like I wanted something a bit more but I couldn't put my finger on what exactly. Turns out what I was missing was the ability to customise the window manager to whatever I want.

    So I finally got round to cloning DWM and adding a few extra bits and pieces to it already and it's been great fun. I was really surprised by how easy it is to apply suckless patches to suckless software. I need to have a browse through the available patches to see if anything tickles my fancy for my build of DWM. I haven't got a lot put into my build at the moment. For DWM itself I've applied the autostart patch so I don't have to manually restart my compositor and nitrogen to display my background. I've also grabbed dwmblocks and written a few scripts for it so I can have a display bar that contains the info I actually care about. It's not the prettiest looking thing at the moment so I'm going to have sort that out soon.

    So far though the transition from i3 to DWM has been really easy, far easier than I thought it was going to be. Customising it is so easy as well. You have to modify the source code if you want to add any other features which does put some people off but the code base is so small and well organised it's a breeze to do. It's even better that I'm comfortable with C which is what DWM is written in! So looking through the source code won't be an intimidating experience.

    Project: Impulse

    In my last blog post I briefly mentioned that I'm working on a new project in Unreal. Well on Friday I managed to finish the to do list for the prototype and got a working build of the project. I don't have any screenshots of the project as of now and that's mainly because the playable version is a rough prototype and it doesn't look nice at all but progress is being made on it. A new to do list has been made and I think it's safe to say the game in now moving into the pre-alpha stage.

    It's nice to see the project slowly coming along but I still need to spend way more time within the Unreal Engine. I'm getting more comfortable with the engine but I still wouldn't say I'm confident with it right now. I know this'll come with time and I follow a few channels on YouTube that do tutorials in Unreal so that should help speed up the process. I've limited myself in this project to try and use blueprint only (Unreal's visual scripting) as much as I can. This is just so I don't waste time trying things within C++ and jump down rabbit holes of trying to implement things that aren't needed or trying to extend the editor when I don't need to. As good as the blueprint system is I do wish there was support for adding components like that via a scripting language like Lua. I'm not sure if the engine does support something like that and I just haven't looked into it but if I had the choice I'd rather write a script than create something with visual scripting. Might be worth looking into in the future but blueprint works perfectly fine for now.
